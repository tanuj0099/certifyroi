# PRD

## Product
Name: CertifyROI

One-line summary:
A premium India-focused certification ROI platform that helps users understand whether a certification is worth the time, fee, and career effort.

## Product intent
The site should feel premium, editorial, intentional, and cohesive across every page.
The landing page is the visual source of truth for the entire system.
All secondary pages should feel like they belong to the same brand world, not like separate templates.

## Core narrative
The product uses the mountain / route / summit metaphor:
- Basecamp = current profile
- Route = chosen certification path
- Summit = measurable ROI outcome

## Primary UX goals
- Make certification ROI feel clear and trustworthy.
- Present tools in an organized, premium, high-signal way.
- Maintain excellent readability in both light and dark mode.
- Ensure all pages share one visual rhythm and one component language.
- Remove all AI-slop patterns, cheap neon effects, and visually generic SaaS styling.

## Source of design truth
The landing page defines:
- heading alignment and hierarchy
- spacing rhythm
- capsule-style button language
- vertical text motif
- editorial premium feel
- restrained accent color usage
- dark/light mode behavior

All other pages should be brought into harmony with this system.

## Key problem areas to solve
1. Top white strip / broken artifact at top of page.
2. Stray `/>` text visible at top-left.
3. Landing hero text visibility problem in light mode over mountain background.
4. Missing landing-page footer.
5. Mode-selection cards/labels are clipping or truncating.
6. Tools page lacks shape, flow, and premium structure.
7. Many internal pages still use inconsistent buttons, boxes, and heading placement.
8. Footer system is incomplete and missing legal/info pages.
9. Some pages still contain cheap-looking glow / neon / blinking visual treatment.
10. Overall site needs one stronger shared system.

## Visual design rules
- Premium, minimal, editorial.
- No glowing neon boxes.
- No flashy cheap gradients unless extremely restrained.
- No blinking decorative cards.
- Buttons across internal pages should inherit the refined capsule system from landing page.
- Green may remain where it indicates positive/high-value/important outcomes.
- Accent colors should retain meaning, not become decoration.
- Headings on internal pages should be centered in the same spirit as landing-page central composition.
- Tools should have clear structure, balance, and flow.
- Add motion only if it serves coherence and polish.
- Vertical text motif may be reused where it improves continuity.

## Functional requirements
- Landing page must have footer restored.
- Footer should include working pages for:
  - Terms and Conditions
  - Cookies
  - Data Privacy
- These legal/support pages should appear in footer only.
- Each of those pages must contain real written content, not placeholders.
- Tools page should be reorganized with better grouping, hierarchy, and visual flow.
- Tool transitions may include subtle morph behavior if it supports clarity and brand consistency.
- Theme system must be fully resolved across all pages and components.

## Content rules
- Do not rewrite content unnecessarily.
- Fix only broken, clipped, nonsensical, placeholder-like, or illegible content.
- Keep important meaning indicators such as green = good/high-value.
- Improve clarity without changing core intent.

## Non-functional requirements
- Excellent readability in light mode and dark mode.
- Responsive across desktop and mobile.
- Consistent spacing, typography, and alignment.
- No layout clipping.
- No stray markup artifacts.
- No blank sections.
- No visual leftovers from removed components.
